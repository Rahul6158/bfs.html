Depth-First Search (DFS) in Data Structures

What is Depth-First Search (DFS)?

Depth-First Search (DFS) is an algorithm for traversing or searching tree or graph data structures. It explores as far as possible along each branch before backtracking, using a stack to keep track of nodes to visit.

Key Concepts of DFS

  • Backtracking: DFS explores a path until it cannot go further, then backtracks to explore other paths.
  • Stack: DFS can be implemented using a stack (either explicitly or through recursion) to keep track of nodes.
  • Path Exploration: DFS can be used to find paths in graphs and trees, including cycles and connected components.

How DFS Works

  1. Start from the Source Node: The algorithm starts from a given node and pushes it onto the stack.
  2. Visit Neighbors: For the node on the top of the stack, visit an unvisited neighbor, push it onto the stack.
  3. Backtrack: If there are no unvisited neighbors, pop the stack and repeat the process until all nodes are visited.

DFS Algorithm

  1. Initialization:
    • Mark all nodes as unvisited.
    • Add the source node to the stack and mark it as visited.
  2. While the stack is not empty:
    • Pop the top node from the stack.
    • Visit all its adjacent nodes. If a node has not been visited, mark it as visited and push it onto the stack.
  3. Repeat until the stack is empty.

DFS Example (Graph)

Consider the following graph: 

    A
   / \
  B   C
 / \
D   E

- Start from node A. - DFS first visits A, then goes to B, then to D, backtracks to B, visits E, and finally backtracks to A, and visits C.

The DFS traversal order for this graph is: A → B → D → E → C.

Applications of DFS

  • Path Finding: DFS can be used to find a path between two nodes in a graph.
  • Topological Sorting: DFS is used in topological sorting of directed acyclic graphs (DAGs).
  • Cycle Detection: DFS can detect cycles in directed and undirected graphs.
  • Solving Puzzles and Games: DFS can explore possible moves in puzzles or games (like mazes).
